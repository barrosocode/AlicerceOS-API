## 1. Resumo técnico implementado

### Infraestrutura (Docker)

- **Dockerfile:** imagem base `python:3.12-slim`, dependência `libpq5`, instalação de dependências via `requirements.txt`, `WORKDIR /app`, exposição da porta 8000.
- **docker-compose.yml:** serviço **`db`** com **PostgreSQL 16** (`postgres:16`, imagem Debian — evita problema de collation na criação do banco de testes observado com variantes Alpine); serviço **`web`** com build da aplicação, `env_file: .env`, volume do código, migrações na subida e `depends_on` com healthcheck do banco.

### Stack de aplicação

- **Django 5.x** com projeto em **`config/`** (`settings`, `urls`, `wsgi`, `asgi`).
- **Django REST Framework** configurado com autenticação JWT.
- **djangorestframework-simplejwt:** endpoints `POST /api/auth/token/` e `POST /api/auth/token/refresh/` em `config/urls.py`.
- **Banco:** PostgreSQL via variáveis `DB_NAME`, `DB_USER`, `DB_PASSWORD`, `DB_HOST`, `DB_PORT` (e `SECRET_KEY`, `DEBUG` no ambiente).

### Apps Django

- **`apps/accounts`:** modelo **`User`** customizado (`AbstractUser`), login por **e-mail** (`USERNAME_FIELD = "email"`, `username = None`), papéis **Admin / Gestor / Colaborador**, **`UserManager`** para criação por e-mail; admin registrado.
- **`apps/core`:** modelos de domínio alinhados a `docs/specs/datamodel_spec.md` — **Cliente**, **Parceiro**, **Projeto**, **Gasto** (este último necessário para testar as regras de integridade da spec); `Gasto.clean()` valida mão de obra com parceiro obrigatório; FK de gasto em projeto com `SET_NULL` na exclusão do projeto.

### Dependências principais

- `Django`, `djangorestframework`, `djangorestframework-simplejwt`, `psycopg[binary]`.

## 3. Comandos de execução

Na raiz do repositório **AlicerceOS-API**:

```bash
docker compose build
docker compose up
```

Criar superusuário (interativo):

```bash
docker compose run --rm web python manage.py createsuperuser
```

Rodar migrações e testes (exemplo):

```bash
docker compose run --rm web sh -c "python manage.py migrate --noinput && python manage.py test apps.core.tests"
```

**Nota:** o ficheiro `.env` deve definir `DB_HOST=db` quando a API corre dentro do Compose; para uso local da API fora do container, usar `DB_HOST=localhost` e manter o Postgres acessível (por exemplo só `docker compose up -d db`).
