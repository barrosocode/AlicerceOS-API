## 1. Resumo técnico

### Domínio (`apps/core`)

- **`Projeto`:** novos campos `descricao_escopo` (`TextField`, opcional) e `data_aprovacao` (`DateField`, opcional), mantendo `orcamento_estimado` e demais campos da spec.
- **`OrcamentoItem`:** FK `projeto` (CASCADE), FK `parceiro` (opcional, PROTECT), `descricao`, `quantidade`, `valor_unitario`, `tipo` (`material` / `mao_de_obra`). Validação: **mão de obra exige parceiro**. Property `subtotal` = `quantidade × valor_unitario`.
- **`Projeto.valor_total_orcamento_itens`:** property que agrega `Sum(quantidade × valor_unitario)` dos itens (RF05).
- **`RelatorioAtividade`:** FK `projeto`, `tipo` (`diario` / `semanal`), `descricao`, `fotos_url` (`TextField` para URLs/JSON conforme operação).

### Integração cross-app (Financeiro)

- Em **`apps/financeiro/models.Gasto.clean()`:** se `projeto` está em status **`orcamento`**, lança `ValidationError` — **não permite gastos reais** até o projeto estar em execução (ativado).

### Permissões de status

- **`apps/core/project_permissions.py`:** `assert_usuario_pode_definir_status_ativo(usuario)` — apenas **`admin`**, **`gestor`** ou **superusuário** podem concluir a transição para **`ativo`**.
- **`ProjetoSerializer.update`:** ao mudar de outro status para **`ativo`**, chama a asserção acima; erros Django são convertidos para `serializers.ValidationError` (HTTP 400 na API).

### API (`/api/`)

- **`GET/PATCH /api/projetos/`** e **`GET/PATCH /api/projetos/{id}/`** — listagem e detalhe com campos calculados:
  - `valor_total_orcamento_itens` (property),
  - `gastos_reais_total` (soma de `financeiro.Gasto` do projeto),
  - `progresso_barra_percentual` (mínimo entre 100% e gastos/ `orcamento_estimado` × 100; `null` se base ≤ 0),
  - `percentual_gasto_sobre_orcamento` (sem teto, para visibilidade de estouro).
- **`/api/orcamento-itens/`** — CRUD completo; filtro opcional `?projeto=<id>`.
- **`/api/relatorios-atividade/`** — CRUD com filtro opcional `?projeto=<id>` (RF06, base para diário de obra).

### Migrations

- **`core.0003_projeto_orcamento_relatorio`:** campos em `Projeto` + modelos `OrcamentoItem` e `RelatorioAtividade`.
